## 技能概述

本技能提供 A 股股票智能筛选能力，帮助用户快速找到符合特定条件的股票。通过自然语言查询，支持多种筛选维度的组合，为投资决策提供数据支持。

**适用场景**：
- 股票筛选（按涨跌幅、估值、财务指标等）
- 股票排行（涨幅前列、市值最大等）
- 技术形态筛选（均线多头、突破新高等）
- 多条件组合筛选
- 行业概念筛选

---

## 核心功能

### 支持的筛选维度

| 维度 | 说明 | 示例查询 |
|------|------|------|
| **行情指标** | 按股价、涨跌幅、成交量、换手率等筛选 | 股价小于10元、涨幅超过5%、放量上涨 |
| **技术形态** | 按均线多头、突破新高、K线形态等筛选 | 均线多头排列、突破半年新高、MACD金叉 |
| **财务指标** | 按营收、利润、PE、PB、ROE等筛选 | 市盈率低于20、净利润增长超过30% |
| **估值指标** | 按市值、PE、PB、PEG等筛选 | 市值小于50亿、PB低于2 |
| **行业概念** | 按行业分类、概念题材筛选 | 科技股、医药股、新能源概念 |
| **资金流向** | 按主力资金、北向资金等筛选 | 主力净流入、北向资金持续流入 |
| **多条件组合** | 多个条件组合筛选 | 涨幅超过5%且市盈率低于20的科技股 |

### 核心能力

1. **自然语言理解**：将用户的口语化查询转换为标准筛选条件
2. **多维度筛选**：支持行情、财务、技术、资金等多维度组合
3. **实时数据**：基于通达信实时行情数据
4. **智能改写**：自动优化查询条件，提高查询成功率

- **通达信K线 (tdx_kline)**：技术分析数据补充

| 工具名称 | 用途 | 使用场景 |
|----------|------|----------|
| tdx_screener | 选股查询 | 必须调用 |
| tdx_quotes | 获取实时行情 | 需要扩展数据时调用 |
| tdx_kline | 获取K线数据 | 技术形态分析时调用 |
| wenda_news_query | 获取相关资讯 | 需要资讯背景时调用 |
| wenda_report_query | 获取研报观点 | 需要专业分析时调用 |

## 使用流程

### 流程图

```
用户输入 → Query改写 → 调用接口 → 数据扩展 → 格式化输出
```

### 详细步骤

#### 步骤 1：接收用户查询

接收用户的自然语言选股请求，分析用户意图和筛选条件。

**用户输入示例**：
- "涨幅超过5%的科技股"
- "市盈率低于20且市值大于100亿"
- "均线多头排列的股票"
- "帮我选些涨得好的科技股"

#### 步骤 2：Query 改写与标准化

将用户的口语化表达改写为标准的金融查询语句。

**改写规则**：
- 保留核心意图（涨幅、估值、行业等）
- 将口语转为标准术语
- 适当简化复杂条件
- 保持原意不变

**标准化映射表**：

| 用户输入 | 标准化 Query |
|---------|---------------|
| 帮我选些涨得好的科技股 | 涨幅超过5%的科技股 |
| 便宜且业绩好的股票 | 市盈率低于20且净利润增长超过30% |
| 放量突破的股票 | 成交量放大且突破新高的股票 |
| 涨得很猛的 | 涨幅超过8% |
| 价格不贵的 | 股价低于20元 |

#### 步骤 3：调用数据接口

使用 `tdx_screener` 工具查询股票数据：

```bash
tdx_screener message="<标准化的Query>" rang="AG"
```

**分页支持**：
- 默认返回第1页
- 每页最多10条记录
- 可通过 `pageNo` 和 `pageSize` 参数控制

#### 步骤 4：数据扩展决策（可选）

根据数据完整性决定是否需要补充信息：

**需要扩展背景信息的场景**：
- 用户请求了深度分析
- 查询涉及技术形态，需要K线数据
- 查询涉及市场热点，需要资讯补充

**扩展方式**：
- 调用 `tdx_quotes` 获取实时行情数据（价格、涨跌幅、成交额等）
- 调用 `tdx_kline` 获取K线数据（趋势、形态分析）
- 调用 `wenda_news_query` 获取相关资讯
- 调用 `wenda_report_query` 获取研报观点

#### 步骤 5：格式化输出

组织语言回答用户，确保结果清晰易懂。

**输出格式要求**：
- 使用表格展示股票列表
- 说明数据来源（通达信问小达）
- 如果改写了问句，需说明最终使用的查询条件
- 如有扩展信息，提供简要分析

## 接口参数说明

### tdx_screener 参数

| 参数名 | 类型 | 必填 | 说明 | 示例 |
|--------|------|--------|------|------|
| message | string | 是 | 自然语言查询语句 | "涨幅超过5%的科技股" |
| rang | string | 否 | 市场范围，默认"AG" | "AG" |
| pageNo | string | 否 | 页码，默认"1" | "2" |
| pageSize | string | 否 | 每页条数，默认"10" | "20" |

### rang 参数可选值

| 值 | 说明 |
|-----|------|
| AG | A股（默认） |
| JJ | 基金 |
| ZS | 指数 |
| ZG-JJJL | 基金经理 |
| GG-GP | 港股 |

## 注意事项

### 重要提示

1. **数据来源标注**
   - 所有引用通达信数据的地方，必须强调**数据来源于通达信问小达**
   - 避免用户对数据来源产生混淆

2. **数据有效性说明**
   - 股票数据为实时行情，可能存在延迟
   - 财务数据有披露周期，通常滞后于行情
   - 某些指标可能需要特定条件才能计算

3. **查询限制**
   - 单次查询最多返回一定数量的结果
   - 复杂条件可能需要分步骤查询
   - 部分股票可能停牌或退市，数据可能缺失

4. **分页说明**
   - 默认返回第一页（前10条）
   - 如需更多结果，可请求翻页
   - 建议先浏览第一页再决定是否继续

5. **性能优化**
   - 避免过于复杂的查询条件
   - 多条件组合时建议分步查询
   - 避免频繁重复相同查询

### 风险提示

1. **筛选结果仅供参考**
   - 筛选结果不构成投资建议
   - 仅供参考和学习使用

2. **独立判断重要性**
   - 用户需根据自身情况独立判断
   - 结合自身风险承受能力决策

3. **投资风险提示**
   - 股票投资有风险，可能损失本金
   - 历史表现不代表未来收益

4. **分散投资建议**
   - 建议合理分散，避免单一持仓
   - 控制单只股票持仓比例

5. **持续学习**
   - 建议持续学习投资知识
   - 提升自身的判断能力

## 错误处理

### 常见错误场景

| 错误类型 | 处理方式 | 用户提示 |
|----------|----------|----------|
| API 调用失败 | 记录日志，显示错误信息 | 提示用户稍后重试 |
| 无数据返回 | 检查查询条件，逐步放宽 | 提示条件可能过于严格，建议放宽条件或访问通达信客户端 |
| 股票代码错误 | 验证代码格式和有效性 | 提示检查股票代码是否正确 |
| 网络超时 | 检查网络连接状态 | 提示检查网络连接，建议重试 |
| 条件过于复杂 | 拆分为多个简单条件 | 建议分步查询 |
| 参数错误 | 验证参数完整性和格式 | 提示用户检查查询条件是否清晰 |

### 重试策略

**重试规则**：
- **最大重试次数**：2次
- **重试间隔**：1秒
- **条件放宽策略**：每次重试适当放宽筛选条件
- **降级方案**：重试失败后引导用户使用通达信客户端查询

### 用户引导与降级方案

当无法获取数据或查询失败时，提示用户：

```
抱歉，未能获取到符合条件的数据。建议：
1. 检查网络连接是否正常
2. 尝试放宽筛选条件
3. 访问通达信客户端进行查询

数据来源：通达信
客户端下载：https://www.tdx.com.cn/
```
