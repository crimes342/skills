## 技能概述

本技能提供ETF智能筛选能力，通过自然语言查询支持多种筛选维度，帮助用户快速找到符合条件的ETF产品。

**适用场景**：
- ETF筛选（按涨跌幅、规模、跟踪指数等）
- ETF排行（涨幅前列、规模最大等）
- 多条件组合筛选
- 指数产品查找
- 风格类型筛选

---

## 核心功能

### 支持的筛选维度

| 维度 | 说明 | 示例查询 |
|------|------|------|
| **行情指标** | 按价格、涨跌幅、成交量、换手率等筛选 | 价格小于2元、涨幅超过3%、放量上涨 |
| **跟踪指数** | 按跟踪的指数筛选 | 沪深300ETF、中证500ETF、上证50ETF |
| **基本指标** | 按估值、费率、跟踪误差等筛选 | 市盈率低于15、费率低于0.5% |
| **规模指标** | 按资产规模、份额变化等筛选 | 规模大于10亿、份额增长 |
| **风格类型** | 按投资风格筛选 | 成长风格、价值风格、平衡风格 |
| **多条件组合** | 多个条件组合筛选 | 沪深300且涨幅超过3%且规模大于50亿 |
| **主题类型** | 按投资主题筛选 | 科技主题、医药主题、新能源主题 |

### 核心能力

1. **自然语言理解**：将用户的口语化查询转换为标准筛选条件
2. **多维度筛选**：支持行情、规模、风格等多维度组合
3. **实时数据**：基于通达信实时行情数据
4. **智能改写**：自动优化查询条件，提高查询成功率
5. **指数匹配**：支持指数名称模糊匹配和精确查询

## 使用流程

### 流程图

```
用户输入 → Query改写 → 调用接口 → 数据扩展 → 格式化输出
```

### 详细步骤

#### 步骤 1：接收用户查询

接收用户的自然语言ETF筛选请求，分析用户意图和筛选条件。

**用户输入示例示例**：
- "涨幅超过3%的ETF"
- "沪深300的ETF"
- "规模大于50亿的ETF"
- "帮我选一个科技主题的ETF"
- "价格便宜且流动性好的ETF"

#### 步骤 2：Query 改写与标准化

将用户的口语化表达改写为标准的金融查询语句。

**改写规则**：
- 保留核心意图（指数、规模、风格等）
- 将口语转为标准术语
- 适当简化复杂条件
- 保持原意不变

**标准化映射表**：

| 用户输入 | 标准化 Query |
|---------|---------------|
| 帮我选一个ETF | ETF有哪些 |
| 给我推荐个ETF | ETF有哪些 |
| 涨得好的ETF | 涨幅前10的ETF |
| 涨幅最大的前3个 | 涨幅前3的ETF |
| 规模大的ETF | 规模前10的ETF |
| 沪深300的基金 | 沪深300ETF |
| 沪深300涨幅前3 | 沪深300ETF按涨幅排名前3 |

**⚠️ 关键：排序查询与条件查询的区别**

| 类型 | 表达方式 | 示例 | 含义 |
|------|----------|------|------|
| **排序查询** | `前N` | "涨幅前3"、"规模前10" | 按指标排序，取最大的N个 |
| **条件查询** | `超过/小于/大于` | "涨幅超过3%"、"规模大于50亿" | 按阈值筛选，返回符合条件的所有结果 |

**常见混淆场景**：
- ❌ "涨幅超过3%" ≠ "涨幅前3"
- ✅ "涨幅超过3%" = 返回所有涨幅>3%的ETF
- ✅ "涨幅前3" = 返回涨幅最大的3只ETF

**思维链拆解**：
- **单次查询**：如果用户问题可以直接用单个 query 回答，直接进入下一步
- **多次查询**：如果用户问题涉及多个独立的问句，需要拆分为多个标准 query 分别调用接口

#### 步骤 3：调用数据接口

使用 `tdx_screener` 工具查询ETF数据：

```bash
tdx_screener message="<标准化的Query>" rang="JJ"
```

**分页支持**：
- 默认返回第1页
- 每页最多10条记录
- 可通过 `pageNo` 和 `pageSize` 参数控制

#### 步骤 4：数据扩展决策（可选）

根据数据完整性决定是否需要补充信息：

**需要扩展背景信息的场景**：
- 用户请求了深度分析
- 查询涉及趋势分析，需要K线数据
- 查询涉及市场热点，需要资讯补充

**扩展方式**：
- 调用 `tdx_quotes` 获取实时行情数据（价格、涨跌幅、成交额等）
- 调用 `tdx_kline` 获取K线数据（趋势、形态分析）
- 调用 `wenda_news_query` 获取相关资讯

#### 步骤 5：格式化输出

组织语言回答用户，确保结果清晰易懂。

**输出格式要求**：
- 使用表格展示ETF列表
- 说明数据来源（通达信问小达）
- 如果改写了问句，需说明最终使用的查询条件
- 如有扩展信息，提供简要分析

| 参数名 | 类型 | 必填 |说明 | 示例 |

| 值 | 说明 |

## 注意事项

### 重要提示

1. **数据来源标注**
   - 所有引用通达信数据的地方，必须强调**数据来源于通达信问小达**
   - 避免用户对数据来源产生混淆

2. **数据有效性说明**
   - ETF数据为实时行情，可能存在延迟
   - 基金规模和份额数据可能有更新周期
   - 某些指标可能需要特定条件才能计算

3. **查询限制**
   - 单次查询最多返回一定数量的结果
   - 复杂条件可能需要分步骤查询
   - 部分ETF可能暂停申赎或退市

4. **分页说明**
   - 默认返回第一页（前10条）
   - 如需更多结果，可请求翻页
   - 建议先浏览第一页再决定是否继续

5. **性能优化**
   - 避免过于复杂的查询条件
   - 多条件组合时建议分步查询
   - 避免频繁重复相同查询

6. **ETF特性说明**
   - ETF可在交易所买卖，流动性关注成交量和换手率
   - ETF净值与交易价格可能有微小差异
   - 注意跟踪误差和费率对长期收益的影响

### 风险提示

1. **筛选结果仅供参考**
   - 筛选结果不构成投资建议
   - 仅供参考和学习使用

2. **独立判断重要性**
   - 用户需根据自身情况独立判断
   - 结合自身风险承受能力决策

3. **投资风险提示**
   - ETF投资有风险，可能损失本金
   - 历史表现不代表未来收益

4. **分散投资建议**
   - 建议合理分散，避免单一持仓
   - 控制单只ETF的持仓比例

5. **费用影响**
   - 注意管理费率对长期收益的侵蚀
   - 建议选择费率较低的ETF

6. **跟踪误差**
   - ETF可能与标的指数存在跟踪误差
   - 选择跟踪误差较小的ETF

## 错误处理

### 常见错误场景

| 错误类型 | 处理方式 | 用户提示 |
|----------|----------|----------|
| API 调用失败 | 记录日志，显示错误信息 | 提示用户稍后重试 |
| 无数据返回 | 检查查询条件，逐步放宽 | 提示条件可能过于严格，建议放宽条件或访问通达信客户端 |
| 基金代码错误 | 验证代码格式和有效性 | 提示检查基金代码是否正确 |
| 网络超时 | 检查网络连接状态 | 提示检查网络连接，建议重试 |
| 条件过于复杂 | 拆分为多个简单条件 | 建议分步查询 |
| 参数错误 | 验证参数完整性和格式 | 提示用户检查查询条件是否清晰 |

### 重试策略

**重试规则**：
- **最大重试次数**：2次
- **重试间隔**：1秒
- **条件放宽策略**：每次重试适当放宽筛选条件
- **降级方案**：重试失败后引导用户使用通达信客户端查询

### 用户引导与降级方案

当无法获取数据或查询失败时，提示用户：

```
抱歉，未能获取到符合条件的ETF数据。建议：
1. 检查网络连接是否正常
2. 尝试放宽筛选条件
3. 访问通达信客户端进行查询

数据来源：通达信
客户端下载：https://www.tdx.com.cn/
```

### ⚠️ 重要：排序查询与条件查询的区别

**排序查询**：
- `涨幅前3` = 涨幅**排名前3**的ETF（按涨幅排序，取最大的3个）
- `规模前10` = 规模**排名前10**的ETF
- `换手率前5` = 换手率**排名前5**的ETF

**条件查询**：
- `涨幅超过3%` = 涨幅数值**超过3%**的所有ETF
- `规模大于50亿` = 规模数值**大于50亿**的ETF
- `换手率超过2%` = 换手率数值**超过2%**的ETF

**示例对比**：
| 用户输入 | 含义 | 返回结果数量 |
|---------|------|------------|
| "涨幅前3的ETF" | 涨幅最大的前3名 | 3只 |
| "涨幅超过3%的ETF" | 涨幅大于3%的所有ETF | 可能很多只 |

**提示**：
- 排序查询用"前N"，条件查询用"超过/小于/大于"
- 混淆会导致查询结果不符合预期

### ETF投资要点

1. **费用敏感性**：长期投资应关注费率，低费率ETF更优
2. **跟踪误差**：选择跟踪误差小的ETF，确保复制指数表现
3. **流动性**：关注成交量和换手率，确保买卖便利
4. **规模效应**：大规模ETF通常流动性更好，跟踪误差更小
5. **持仓透明**：ETF每日公布持仓，便于了解投资组合
